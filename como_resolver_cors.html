Como Resolver o Erro CORS

Você está enfrentando um erro de política CORS (Cross-Origin Resource Sharing) ao tentar acessar o arquivo manifest.json do seu Progressive Web App (PWA).

Erro: Access to internal resource at 'file:///C:/RPJDigitalSites/SALARIO/manifest.json' from origin 'null' has been blocked by CORS policy: Cross origin requests are only supported for protocol schemes: chrome-extension, chrome-untrusted, data, edge, http, https, isolated-app.

Por que isso acontece?

Este erro ocorre porque você está tentando acessar o arquivo manifest.json usando o protocolo file:// (abrindo o arquivo HTML diretamente no navegador), mas os navegadores modernos bloqueiam esse tipo de acesso por questões de segurança.

Como resolver

Para resolver este problema, você precisa servir seus arquivos através de um servidor HTTP. Existem várias maneiras de fazer isso:

Opção 1: Usar o Node.js

Se você tem o Node.js instalado, pode usar o servidor que criamos:

  1. Abra o Prompt de Comando (CMD) ou PowerShell
  2. Navegue até a pasta do seu projeto:
    cd C:\RPJDigitalSites\SALARIO
  3. Execute o servidor Node.js:
    node server.js
  4. Acesse seu aplicativo no navegador através de:
    http://localhost:8080

Opção 2: Usar o http-server

Uma alternativa simples é usar o pacote http-server:

  1. Instale o http-server globalmente (se ainda não tiver):
    npm install -g http-server
  2. Navegue até a pasta do seu projeto:
    cd C:\RPJDigitalSites\SALARIO
  3. Inicie o servidor:
    http-server . --cors
  4. Acesse seu aplicativo no navegador através de:
    http://localhost:8080

Opção 3: Usar extensões de navegador

Para desenvolvimento, você pode usar extensões que desativam temporariamente as restrições de CORS:

  • Para Chrome: CORS Unblock
  • Para Firefox: CORS Everywhere
Nota: Usar extensões para desativar CORS é recomendado apenas para desenvolvimento local, não para produção.

Opção 4: Usar serviços de hospedagem

Para uma solução permanente, hospede seu PWA em um serviço de hospedagem web como:

  • GitHub Pages (gratuito)
  • Netlify (gratuito)
  • Vercel (gratuito)
  • Firebase Hosting (gratuito)

Verificação do manifest.json

Seu arquivo manifest.json parece estar correto. Certifique-se de que todos os caminhos de ícones estão corretos e que os arquivos existem nas localizações especificadas.

Lembre-se: PWAs só funcionam completamente quando servidos através de HTTPS ou HTTP local (localhost).
